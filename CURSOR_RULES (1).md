## Cursor Rules

### Development Rules
Define the delivery stack and guardrails for implementation.

- **Languages to use?**
  The app is developed using JavaScript for the frontend and Python for the backend.

- **Frontend framework preference?**
  React is used as the frontend framework for its efficiency and extensive community support.

- **Styling approach?**
  CSS with a preprocessor like Sass is used for styling, ensuring consistency and ease of maintenance.

- **State management plan?**
  State management is handled using Redux for predictability and debugging ease.

- **Backend or data layer?**
  The backend is built using Flask or Django, chosen for their lightweight nature and rapid development capabilities.

- **APIs to connect and which to mock?**
  APIs for nutrition data and potential health integrations are connected, with mocked APIs used during development for stability.

- **Testing expectations?**
  Unit testing and integration testing are performed using Jest and Pytest, respectively, to ensure code reliability.

- **Security guidelines?**
  Security guidelines follow OWASP recommendations, with HTTPS encryption and secure password hashing.

- **Performance considerations?**
  Performance optimization includes code minification, caching, and efficient database queries.

- **File organization conventions?**
  The project follows a modular structure, with separate folders for frontend, backend, and shared utilities.

- **Commit conventions?**
  Commit messages follow the GitHub guidelines, being concise and descriptive, with issues referenced when applicable.
